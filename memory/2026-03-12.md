## Delta Healing Solutions - ULTIMATE Form Styling v4 (8:18 AM)

**Issue:** v3 CSS still wasn't removing boxes/borders from service buttons and newsletter checkbox. Squarespace's inline styles and specific selectors were overriding our CSS.

**Solution:** Created **ULTRA-CLEAN-FORM.css** with ultra-aggressive selectors

### What Makes v4 "ULTIMATE":

**1. Multiple Option Selectors:**
```css
.form-wrapper .option,
.form-wrapper .field.checkbox .option,
.form-wrapper .field.radio .option,
.form-wrapper .field-element .option,
div.option,            /* ← NEW: catches generic divs */
.option                /* ← NEW: catches any .option */
```

**2. Triple Border Removal:**
```css
border: none !important;
border-width: 0 !important;     /* ← NEW */
box-shadow: none !important;    /* ← NEW */
```

**3. Double Background Removal:**
```css
background: transparent !important;
background-color: transparent !important;  /* ← NEW */
```

**4. Newsletter Block - Multiple Variations:**
```css
.form-wrapper .newsletter-block,
.form-wrapper .newsletter-block .option,
.form-wrapper .newsletter-block label,
.form-wrapper .newsletter-block .field-element,
.newsletter-block,
.newsletter-block .option
```
All set to transparent, no borders, no box-shadow

**5. Fieldset Border Removal:**
```css
.form-wrapper fieldset,
.form-wrapper .fieldset,
fieldset {
  border: none !important;
  border-width: 0 !important;
}
```

**6. Checked State - Sibling Selector:**
```css
.form-wrapper input[type="checkbox"]:checked ~ label
```
Uses `~` (sibling) instead of just `+` (adjacent) to catch more variations

### Why v4 Was Needed:

Squarespace uses:
- Generic `div.option` classes (not caught by `.form-wrapper .option`)
- Both `background` and `background-color` properties
- Both `border` and `border-width` properties
- Box-shadows on some themes
- Inline styles that override basic CSS

v4 casts a much wider net with multiple selector variations and `!important` on everything.
